# React Testing

The goals are to ensure that software follows all requirements

## Types

### Functional Testing

- e2e
- integration
- screenshot
- unit

### Nonfunctional testing

- payload
- security
- regression testing

## Percentage of amount of tests

- e2e - 10%
- integration - 20-30%
- screenshot 70-80%
- unit

## Tools

- webdriverIO - e2e (like cypress, puppeteer, hermione)
- react testing library - integration (react-router-dom + redux)
- storybook + loki - screenshot
- jest - unit

## Jest

- describe - to wrap several tests
