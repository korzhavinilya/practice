### Экосистема shadcn: что там есть, чего нет и с чем это обычно «женят» в продакшене

#### Что обычно берется из shadcn?

Из shadcn стоит брать всё, что касается сложного поведения и доступности (A11y). То, на чем легко сломать зубы при написании с нуля:

- Модалки и Поповеры (Dialog, Popover, Sheet): Ловушки фокуса, закрытие по Esc, блокировка скролла — всё это уже отлажено в Radix UI, на котором базируется shadcn.
- Выпадающие списки (Select, DropdownMenu, Combobox): Навигация стрелками с клавиатуры — это ад для ручной разработки.
- Формы (Form): В shadcn это очень крутая обертка над React Hook Form и Zod. Это стандарт де-факто.
- Аккордеоны и Табы: Просто чтобы не тратить время на анимации и логику переключения.

Что пишется вручную?

- Layout-компоненты: Section, Container, Flex, Grid. Обычно это просто div с классами. Делать для них отдельный shadcn-компонент — оверхед.
- Простые UI-элементы с уникальным дизайном: Если тебе нужна кнопка, которая выглядит как «летающая тарелка» с кучей SVG-анимаций, проще написать её с нуля на Tailwind + Framer Motion.
- Бизнес-компоненты: UserCard, OrderSidebar, ProductGrid. Это не UI-кит, это компоненты твоего приложения. Они используют внутри себя Button или Badge из shadcn, но сами по себе лежат в папке features или entities.

Внешние библиотеки + shadcn

- Таблицы (TanStack Table)

  В shadcn есть раздел Table. Но это просто стилизованные теги `<table>`, `<tr>`, `<td>`.
  - Как делать: Ты устанавливаешь @tanstack/react-table (логика: сортировка, фильтрация, пагинация) и используешь визуальные компоненты от shadcn для рендеринга.
  - В документации shadcn есть огромный раздел "Data Table" — это фактически туториал по их связке.

- Календари и Даты

  Сам компонент Calendar в shadcn — это обертка над библиотекой react-day-picker.
  - Для манипуляций с датами (сравнение, форматирование) всегда добавляй date-fns.
  - Если нужен полноценный DatePicker, ты берешь Popover + Calendar из shadcn.

- Графики (Recharts)

  Недавно в shadcn появился раздел Charts. Это невероятно красивая обертка над Recharts. Раньше графики в вебе всегда выглядели «чужеродно», а теперь они используют те же CSS-переменные, что и остальной твой UI.

#### Стек-рекомендация для реального проекта

| Задача    | Инструмент                                     |
| --------- | ---------------------------------------------- |
| UI Основа | shadcn/ui (Radix UI + Tailwind)                |
| Иконки    | lucide-react (идут в комплекте)                |
| Формы     | react-hook-form + zod                          |
| Таблицы   | tanstack/react-table                           |
| Анимации  | framer-motion (идеально для плавных появлений) |
| Даты      | date-fns + react-day-picker                    |

### Audit Form пример

Компонент Form: В shadcn это не просто тег <form>, а мощный контекст. Обрати внимание на FormField. Он автоматически связывает input с ошибками валидации Zod. Тебе не нужно вручную проверять errors.username.

Date Picker: В shadcn нет одного тега <DatePicker />. Это композиция: Popover (всплывающее окно) + Calendar (логика выбора даты) + Button (триггер). Это дает тебе полный контроль: хочешь — добавь иконку, хочешь — измени формат даты в кнопке.

Chart: Используется новый ChartContainer. Его главная фишка в том, что цвета задаются через CSS-переменные (например, hsl(var(--chart-1))). Это значит, что если пользователь переключит сайт в Dark Mode, график поменяет цвета автоматически вместе со всей темой.

Zod: Валидация происходит на лету. Если пользователь не выбрал дату, Popover подсветится красным, а FormMessage выведет текст из required_error.
